# Sidebar Menu UI Design

## Intent

Improve the sidebar so it feels like an operational control panel instead of a generic list of buttons.

The menu should stay compact for experienced users, but gain clearer hierarchy and location awareness for daily navigation.

## Direction

Use a balanced visual language between dense and guided:

- group headers with subtle separators
- primary items with a quiet surface and stronger active state
- a thin operational rail for active entries
- submenus that feel structurally tied to their parent
- collapsed mode that still communicates hierarchy cleanly

## Visual Rules

- Same surface family across the sidebar, separated with low-contrast borders
- Active state uses a soft background elevation plus a thin left rail
- Hover states rely on small surface shifts instead of heavy color blocks
- Icons sit inside a stable rounded cell to improve scan rhythm
- Group labels use uppercase micro-label styling to organize sections without shouting

## Result

The sidebar should feel more intentional, more readable, and still efficient for repeated operational use.
